My Key Skill

My MQA Key Skill


1. Pemahaman Dasar tentang Pengujian Perangkat Lunak
  • Konsep Dasar : SDLC (Software Development Life Cycle), STLC (Software Testing Life Cycle), dan metodologi Agile/Scrum.
  • Tipe Pengujian : Unit testing, Integration testing, System testing, User Acceptance Testing (UAT), Smoke Testing, dan Regression Testing.
  • Konsep Bug : Memahami siklus hidup bug (bug lifecycle) dan cara melaporkannya dengan jelas.
2. Keterampilan Analisis
  • Kemampuan untuk memahami persyaratan bisnis dan dokumen spesifikasi (Business Requirement Document - BRD, Functional Requirement Document - FRD).
  • Menganalisis kebutuhan pengujian untuk menciptakan skenario pengujian (test scenarios) dan kasus uji (test cases).
3. Penulisan dan Eksekusi Test Case
  • - Menulis test case yang lengkap, jelas, dan mudah dipahami.
  • - Menggunakan teknik desain pengujian seperti Boundary Value Analysis (BVA), Equivalence Partitioning, dan Decision Table Testing.
4. Dokumentasi
  • Membuat laporan pengujian (testing reports/ test case document).
  • Mengelola dokumen seperti log bug, laporan hasil pengujian, dan pelacakan pengujian (traceability matrix).
5. Penggunaan Alat Pengujian Manual
  • Familiar dengan alat pelaporan bug dan manajemen proyek, seperti Jira
6. Pemahaman tentang Basis Data
  • Dasar-dasar SQL untuk memeriksa data di basis data, seperti melakukan query untuk verifikasi data backend.
7. Keterampilan Komunikasi
  • Menyampaikan temuan bug atau masalah teknis dengan jelas kepada pengembang dan pemangku kepentingan non-teknis.
  • Berkolaborasi dengan tim lintas fungsi, termasuk pengembang, manajer proyek, dan desainer.
8. Fokus pada Detail
  • Memiliki ketelitian untuk menemukan bug kecil sekalipun.
  • Melakukan pengujian menyeluruh pada berbagai skenario.
9. Pemahaman Tentang Browser dan Perangkat
  • Pengujian di berbagai browser (cross-browser testing) dan perangkat untuk memastikan kompatibilitas (desktop, mobile, tablet).
10. Kemampuan Exploratory Testing
  • Kreativitas dalam menemukan bug melalui eksplorasi aplikasi tanpa bergantung pada test case yang sudah ada.
